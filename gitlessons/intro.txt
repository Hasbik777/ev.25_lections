GIT - распределенная система контроля версий

Это система для отслеживания и ведение истории изменения файлов, в вашей проекте.

Репозитор - хранилище вашего кода и истории его изменений

GitHub - Веб сайт для размещения git-респозиториев в совместной разработке проектов

Команда Git:
1. git init - она создает новый локальный гит репозиторий у вас на пк, в той директории в которой была прописана команда. (Команда вводиться один раз в начале)
2. git add - команда для добавления файлов в систему отслеживания гита, после этой команды изменения начинают отслеживаться гитом.
git commit -m '<comment>" - как только достигаем определенного момента в разработке, то тогда мы сохраняем и комментируем изменения в локальной репозиторий
4. git remote add - это команда для того что бы связать наш локальный репозиторий с удаленным репозиторием, то есть добавляет подключение к репозиторий на гитхабе
   git remote add <name: origin> <ссылка_на_репо:ssh>
5. git push - команда для отправки всех ваших изменений на локальном репозитории в удаленный репозиторий
   git push origin main (master)
  __________________________________
  1) git init (только 1 раз)
  2) git add .
  3) git commit -m ''
              3.1) git branch -M main (1 раз, для переименования ветки) 
              *) git remote add origin <ссыслка> (1 раз)
  4.*) git push origin main (master)   

6. git pull <origin> <branch> - команда при помощи которой можно стягиваеть изменения с удаленного репозитории

7. git clone <url> - Команда для клонирования уаленного репозитория и всего его содержимого

8. git branch <name> - команда для создание новой ветки 

9. git checkout <name> - команда для переключение ветки

10. git revert - команда для возвращения на определенный коммит, соотвественно для  восстановления предыдущих версиий кода
git revert <ссылка на коммит>

git log - можно посмотреть историию всех коммитов и посмотреть ссылки на коммиты
